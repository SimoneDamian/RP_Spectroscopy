When the Finite State Machine of the Laser Manager (@laser_manager.py) is in the LOCKED state, the lock_check method is triggered at each cycle. It has to send the histories to the GUI that plots them. Now I have to handle these histories. I want a sort of real time (oscilloscope in the roll mode) visualization of the history. Three parameters are fundamental for these plots: the control_signal_history_length parameter that is the length of the history data that comes from the Red Pitaya, the "fast" and the "slow" "history_length" of the gui_visualization advanced settings that controls the length of the history that is elaborated locally and that has to be plotted (@RedPitaya_K_parameters.yaml). So, every cycle of the control_loop in the LOCKED state, the Laser Manager has to get the histories from the Red Pitaya through the interface's get_history method (@interface.py) and has to send the result to the GUI that will plot them in a new LockingMonitorPlotHandler (@plot_panel.py) that, for now, has to show 3 separate plots: one for the "monitor_values", one for the "fast_control_values" and one for the "slow_control_values" and has the same buttons of the MessagePlotHandler of the LOCKED state that is actually used. To the already implemented get_history method of the @interface.py you have to implement a way of obtaining 2 kinds of circular buffers: one has to be "fast" seconds long and the other one should be "slow" seconds long. Everything that is related to the fast control and to the monitor has to be of the first kind, instead, the slow control related things have to be of the second one. These data are the one that the Laser Manager has to send to the GUI to get visualized.
