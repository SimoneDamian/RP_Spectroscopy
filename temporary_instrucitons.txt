----- New parameters page instructions -----
I modified the structure of the writeable parameters in the .yaml file that contains the parameters of the Red Pitaya (you can find an example in @RedPitaya_K_parameters copy.yaml) and I want to update the load_RedPitaya_parameters method inside the @interface.py in order to read it correctly (I simply added more entries for each parameter). The entries are: hardware_name, gui_name, description that are strings, type that can be int, float or bool, initial_value, min, max, that are numbers of the type aforementioned or can also be null, scaling that are, in general, float and units that can be a string or can be null.
I also want to change the visualization of these parameters table in the GUI. The page is the ParametersPage class in the @laser_controller_page.py and the parameters are loaded in the table using the load_parameters method using the entries of the dictionary of the writeable parameters. In this new version I want to have, for each row, these columns: ["Name", "Value", "Units", "Min", "Max"] that are linked to the entries of the dictionary elements [gui_name, value, units, min, max]. As it is now, only the "Value" column is editable by the user.
In addition, I want the user to be able to see the "description" of a parameter when the arrow stays on it.

For the unlock_detection group: the title is the ui_label one. Then I want a drop-down menu for each one of the other elements (events, fast_control, slow_control, monitor_signal), each one named by its gui_name. Inside this drop-down menu I want another drop-down menu for each entry with the associated gui_name. Inside this I want a row with the other informations like a checkbox for the enabled status and, where it is present, a textbox for the threshold or for the num_points.
