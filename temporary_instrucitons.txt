When the check_for_lock_status gives True, the FSM is set on the LOCKED mode that will send to the GUI the package with the infos to show  on the right in the locking monitor.
When the FSM is in the LOCKED state the lock_check method is triggered at each cycle. It has to: check the lock status with the selected methods, send the histories to the GUI. If the lock is lost, the FST should pass in the SWEEP mode.
The history is managed by the LaserManager, It receives the history from the Interface and it fills the actual history that is shown to the user with the appropriate length.
