----- Scan page instructions -----
I want to populate the "Scan" page inside the @laser_controller_page.py left stack. From the top to the bottom of the page. A row with three textboxes for the user to insert the "Start voltage", "End voltage" and the "Number of points". Below these I want, in the same row, the "Start scan" and the "Stop scan" buttons. At the bottom of the page, as for the other pages in the stack, I want the "Back" button.
The user fills the three textboxes and presses the "Start scan" button. This triggers the start_scan method (already implemented) of the LaserManager in @laser_manager.py. This changes the state of the Finite State Machine (already implemented) that sends a packet labeled with the "mode":"SCAN" and that is a dictionary structured as follows: packet = {"mode": "SCAN", "step_index": self.scan_index, "total_steps": len(self.scan_voltages), "current_voltage": target_v, "scan_data": self.scan_results, # Sends all accumulated data "latest_sweep": current_sweep   # Sends just the newest trace}. When the "Start scan" button is pressed, I want the "Start scan" and the "Back" buttons to be deactivated. When, during the scan, the user presses the "Stop scan" button, this triggers the stop_scan method in @laser_manager.py that changes the state of the Finite State Machine to "IDLE" (already implemented) and then the "Start sweep" and the "Back" buttons have to be re-enabled. When the "Back" button is pressed, the left part of the window must return to the main menu and the start_sweep method inside @laser_manager.py has to be triggered.
When the GUI receives the signal with the package with the "mode":"SCAN", the visualization of the plot panel (details in @plot_panel.py and implementation in the page in @laser_controller_page.py) has to look like this: A scrollable element organized as a column of plots of the "scan_data" of the different "current_voltage" of the scan. In each single plot I want as title the "current_voltage" and the plot of the "scan_data" (that contains "x" and "error_signal"). Each time the GUI receives a new package it has to plot only the last plot, keeping the old one visualized. At the bottom of the page I want a status bar that shows to the user the percentage of the scan (using the "step_index" and the "total_steps" entries of the package).
